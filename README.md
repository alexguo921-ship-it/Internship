# AI内部竞聘评审系统 · 运行说明

## 快速启动

```bash
# 1. 进入目录
cd 腾讯作业

# 2. 安装依赖（建议 Python 3.9+）
pip install -r requirements.txt

# 3. 启动内部 HR 管理系统（终端窗口 A）
streamlit run app.py --server.port 8503

# 4. 启动外部候选人门户（终端窗口 B）
streamlit run candidate_portal.py --server.port 8504
```

| 应用 | 地址 | 用途 |
|------|------|------|
| 内部 HR 系统 | http://localhost:8503 | HR / 面试官使用，含评审、复核、报告生成、申诉管理 |
| 外部候选人门户 | http://localhost:8504 | 候选人使用，查询报告 / 提交申诉 / 查询申诉进度 |

两个应用通过 `portal_data/` 目录下的共享文件自动同步数据，无需额外配置。

---

## 功能模块（内部系统 · port 8503）

| 标签页 | 演示内容 |
|--------|---------|
| 📊 总览 & 监控 | 360人评分分布、区间占比、外部门户实时流量监控、两周甘特图 |
| 🌐 候选人门户 | 外部独立站点入口 + 数据同步说明 + 外部流量汇总 |
| 📤 简历导入 | 批量 PDF / DOCX 简历上传，AI 自动解析 |
| 👤 员工信息与对比 | 可筛选/搜索的评审表格、个人雷达图、多人横向对比 |
| 🔍 人工复核 | 模拟 HR 对黄区候选人进行审批操作 |
| 📋 能力发展报告 | 一键生成个性化报告（支持 Claude API 或智能模板），生成后自动同步到外部门户 |
| 📣 申诉管理 | 汇总内部录入 + 外部门户提交的所有申诉，HR 可更新状态和回复 |
| 📈 效果评估 | KPI 仪表盘、AI赋能前后对比、离职率逻辑路径 |

## 功能模块（外部候选人门户 · port 8504）

| 标签页 | 演示内容 |
|--------|---------|
| 🔍 查询我的报告 | 按姓名/工号查询评分，查看 HR 已生成的能力发展报告 |
| 📣 提交申诉 | 填写申诉表单，AI 自动分类 + 初步回复，写入共享文件 |
| 📌 查询申诉进度 | 按申诉编号查询 HR 处理状态和回复意见 |

---

## 数据共享机制

```
portal_data/
  ├── candidates.csv    内部系统启动时自动导出（外部门户只读）
  ├── reports.json      HR生成报告后写入（外部门户可展示）
  ├── appeals.json      外部门户写入（内部申诉管理读取+更新）
  └── analytics.json    外部门户访问记录（内部监控读取）
```

---

## Claude API 配置（可选）

在左侧边栏「Claude API」栏输入 API Key，即可使用真实 AI 生成能力报告。

未配置时自动使用智能模板报告，同样可完整演示功能。

- 评分权重、复核阈值可在左侧边栏实时调整，图表自动更新
